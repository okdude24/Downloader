# Downloader

این مخزن برای زمانی ساخته شده که دسترسی به اینترنت بین‌الملل در ایران محدود یا قطع است، اما هنوز چند سرویس مثل GitHub در دسترس هستند. ایده این پروژه از ریپوی [ali25az/ir-downloader](https://github.com/ali25az/ir-downloader) گرفته شده و در این نسخه، علاوه بر دانلود فایل‌های معمولی، یک ورک‌فلو جداگانه برای دانلود از یوتیوب هم اضافه شده است.

ایده کلی ساده است:

1. ورک‌فلو روی GitHub Actions اجرا می‌شود.
2. فایل یا ویدیوی موردنظر روی سرور GitHub دانلود می‌شود.
3. خروجی به فایل فشرده چندتکه تبدیل می‌شود.
4. تکه‌ها داخل خود مخزن commit و push می‌شوند.
5. کاربر می‌تواند بعداً همان تکه‌ها را از GitHub دانلود و روی سیستم خودش extract کند.

## شروع سریع

### 1. اول پروژه را Fork کنید

برای استفاده از این پروژه، اول باید همین ریپازیتوری را به اکانت خودتان fork کنید. چون ورک‌فلوها فایل دانلودشده را داخل مخزن شما commit و push می‌کنند، بهتر است همه چیز روی فورک خودتان اجرا شود.

### 2. دسترسی نوشتن را فعال کن (فقط یک بار!)

این یک مرحله بسیار کوتاه است و اگر انجام ندهی، فرآیند دانلود کار نخواهد کرد. چون فایل دانلود شده باید در ریپازیتوری تو ذخیره شود و عملیات `Write` انجام می‌شود، باید این اجازه را فعال کنی.

مراحل:

1. در صفحه ریپازیتوری fork شده خودت، به تب `Settings` برو.
2. از منوی سمت چپ، وارد `Actions` و بعد `General` شو.
3. به پایین اسکرول کن تا به بخش `Workflow permissions` برسی.
4. گزینه `Read and write permissions` را انتخاب کن.
5. روی `Save` بزن.

## ورک‌فلوها

### 1. دانلود فایل مستقیم

فایل ورک‌فلو: [`.github/workflows/smart-downloader.yml`](.github/workflows/smart-downloader.yml)

این ورک‌فلو برای لینک مستقیم فایل است؛ مثل:

- فایل ویدیویی
- فایل صوتی
- تصویر
- PDF و فایل‌های متنی/آفیس
- سایر فایل‌ها

ورودی‌ها:

- `file_url`: لینک مستقیم فایل
- `password`: رمز اختیاری برای فایل فشرده
- `split_size`: اندازه هر تکه. پیش‌فرض `95m`

رفتار ورک‌فلو:

- فایل با `curl` دانلود می‌شود.
- نوع فایل تشخیص داده می‌شود.
- فایل داخل یکی از دسته‌های `video`، `audio`، `image`، `document` یا `other` قرار می‌گیرد.
- خروجی با `7z` فشرده و به چند بخش تقسیم می‌شود.
- فایل‌ها داخل مخزن push می‌شوند.

مسیر ذخیره خروجی:

```text
downloads/<category>/<file-name>/archive.7z.001
downloads/<category>/<file-name>/archive.7z.002
...
```

### 2. دانلود از یوتیوب

فایل ورک‌فلو: [`.github/workflows/download-youtube.yml`](.github/workflows/download-youtube.yml)

این ورک‌فلو برای دانلود یک ویدیوی یوتیوب و ذخیره آن در مخزن است.

ورودی‌ها:

- `video_url`: لینک ویدیوی یوتیوب
- `quality`: کیفیت دانلود
- `split_size`: اندازه هر تکه. پیش‌فرض `95m`
- `password`: رمز اختیاری برای فایل فشرده

کیفیت‌های قابل انتخاب:

- `best`
- `1080p`
- `720p`
- `480p`
- `360p`
- `audio-only`

رفتار ورک‌فلو:

- `yt-dlp` به صورت مستقیم از GitHub نصب می‌شود.
- عنوان ویدیو خوانده و برای نام پوشه تمیزسازی می‌شود.
- ویدیو یا صدا دانلود می‌شود.
- خروجی فشرده و چندتکه می‌شود.
- فایل‌ها داخل مخزن push می‌شوند.

مسیر ذخیره خروجی:

```text
downloads/youtube/<quality-or-audio>/<video-title>/archive.7z.001
downloads/youtube/<quality-or-audio>/<video-title>/archive.7z.002
...
```

## روش استفاده

### اجرای دانلود فایل مستقیم

1. وارد تب `Actions` در گیت‌هاب شوید.
2. ورک‌فلو `📦 دانلود و ذخیره در مخزن` را انتخاب کنید.
3. روی `Run workflow` بزنید.
4. مقدار `file_url` را وارد کنید.
5. در صورت نیاز، `password` و `split_size` را تنظیم کنید.
6. اجرای ورک‌فلو را شروع کنید.

### اجرای دانلود یوتیوب

1. وارد تب `Actions` شوید.
2. ورک‌فلو `🎬 دانلود ویدیو یوتیوب و ذخیره در مخزن` را انتخاب کنید.
3. روی `Run workflow` بزنید.
4. لینک ویدیو را در `video_url` وارد کنید.
5. کیفیت موردنظر را از `quality` انتخاب کنید.
6. در صورت نیاز، `password` و `split_size` را تنظیم کنید.
7. اجرای ورک‌فلو را شروع کنید.

## آموزش تنظیم کوکی یوتیوب

یک مرحله جدید برای مدیریت کوکی اضافه شده است. بعضی ویدیوهای یوتیوب بدون کوکی دانلود نمی‌شوند. این مخزن از Secret با نام `YT_COOKIES` پشتیبانی می‌کند.

### 1. استخراج کوکی از مرورگر

در Chrome یا Firefox افزونه `Get cookies.txt LOCALLY` را نصب کنید، به یوتیوب بروید و کوکی را export بگیرید.

### 2. ذخیره در GitHub Secret

در مخزن خود بروید به:

`Settings` → `Secrets and variables` → `Actions` → `New repository secret`

مقدارها:

- نام: `YT_COOKIES`
- مقدار: محتوای کامل فایل `cookies.txt`

اگر این secret تنظیم نشده باشد، ورک‌فلو باز هم اجرا می‌شود، اما ممکن است بعضی ویدیوها دانلود نشوند.

## بازیابی فایل‌ها

بعد از پایان ورک‌فلو، فایل‌ها داخل همین مخزن ذخیره می‌شوند. برای استفاده:

1. همه تکه‌های آرشیو را دانلود کنید.
2. فقط فایل اول یعنی `archive.7z.001` را با `7-Zip` یا `p7zip` باز کنید.
3. اگر رمز گذاشته‌اید، همان رمز را وارد کنید.
4. بقیه تکه‌ها به صورت خودکار شناسایی می‌شوند.

نمونه استخراج در لینوکس:

```bash
7z x archive.7z.001
```

## نکات مهم

- مقدار پیش‌فرض `split_size` برابر `95m` است تا خروجی به تکه‌های کوچک‌تر تقسیم شود.
- اگر چند بار یک فایل یا ویدیو را با همان نام اجرا کنید، آرشیوهای قبلی همان پوشه حذف و با خروجی جدید جایگزین می‌شوند.
- نام فایل‌ها و عنوان ویدیوها برای جلوگیری از خطا در مسیرها، تا حدی پاک‌سازی می‌شوند.
- برای دانلود یوتیوب، پلی‌لیست دانلود نمی‌شود و فقط همان ویدیوی ارسالی پردازش می‌شود.
- اگر push با خطا مواجه شد، تنظیمات مجوز GitHub Actions را بررسی کنید تا ورک‌فلو اجازه `Read and write` روی محتویات مخزن داشته باشد.
